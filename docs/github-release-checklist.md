# GitHub Release Checklist

Use this checklist before making the repository public or pinning it on your profile.

## Repository content

- [ ] README is clean and free of placeholders you no longer need
- [ ] documentation files are updated to reflect the actual VM state
- [ ] notes contain no accidental secrets or sensitive internal details
- [ ] screenshots folder contains the final selected evidence
- [ ] no temporary files, exports, or editor artifacts are committed

## Presentation quality

- [ ] repository description is filled in on GitHub
- [ ] topics are added on GitHub
- [ ] social preview image is added
- [ ] README renders correctly on GitHub
- [ ] screenshot links work
- [ ] project title and summary are clear to a reviewer in under 20 seconds

## Suggested GitHub description

`Defensive Linux hardening lab documenting secure baseline configuration, SSH hardening, UFW, fail2ban, AppArmor review, verification steps, and recovery notes on Ubuntu Server.`

## Final validation

- [ ] collect-baseline script runs cleanly in the VM
- [ ] verify-hardening script runs cleanly in the VM
- [ ] README claims match the screenshots
- [ ] tradeoffs and recovery notes are honest and specific
- [ ] the repo reads like a completed lab, not an unfinished outline
