# Screenshots Checklist

Use this checklist before publishing the repository.

## Required screenshots

- [ ] `baseline-system-info.png`
- [ ] `ssh-config-test.png`
- [ ] `ufw-status.png`
- [ ] `fail2ban-status.png`
- [ ] `apparmor-status.png`
- [ ] `verification-script.png`

## Capture quality rules

- [ ] crop the image so the relevant evidence is clearly visible
- [ ] avoid unnecessary desktop clutter
- [ ] keep terminal text readable
- [ ] avoid leaking private IPs you do not want public
- [ ] avoid showing unrelated browser tabs or local files
- [ ] make sure the screenshot matches the documented final state

## Recommended capture order

1. baseline system information
2. SSH configuration test
3. UFW status
4. fail2ban status
5. AppArmor status
6. verification script output

## Final check

- [ ] all screenshot filenames match the README exactly
- [ ] screenshots render correctly on GitHub
- [ ] screenshots support the claims made in the README
