# Security Policy

## Scope

This repository is an educational Linux hardening lab created for learning, documentation, portfolio review, and technical discussion.

It is **not intended for production deployment** and should not be treated as a production-ready security baseline.

## Supported Versions

Only the latest version on the `main` branch is considered in scope for security-related review.

## Reporting a Vulnerability

If you notice a security issue, implementation weakness, or documentation problem, please **do not open a public issue containing exploit details**.

Instead, report it privately by email:

**emirrcanitez@icloud.com**

Please include:
- a short description of the issue
- affected file, script, or documented step
- reproduction notes
- expected impact
- optional mitigation suggestions

## Response Expectations

This repository is maintained as a student project and portfolio lab, so response times may vary.

Reasonable reports will be reviewed as soon as possible.

## Responsible Disclosure Preference

Please prefer responsible disclosure:
- avoid publishing sensitive exploit details before review
- avoid including destructive payloads
- avoid sharing secrets, credentials, or personal data
- keep proof-of-concept details limited and safe

## Notes

Because this is a defensive educational project:
- some controls are intentionally simplified
- no guarantee is made for production suitability
- settings should always be reviewed before being reused elsewhere
